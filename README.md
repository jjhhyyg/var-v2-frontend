# VAR Molten Pool Analysis Desktop Frontend

[简体中文](README.zh.md) | English

> The current frontend targets the Tauri desktop app. Runtime communication uses Tauri commands and Tauri events; the Windows analysis stack is distributed as a separate CUDA runtime zip.

## Responsibilities

- Provide the Nuxt 4 + Vue 3 + Nuxt UI desktop interface
- Manage the media library, task database, task queue, and settings through Tauri commands
- Support batch video import, FIFO queueing, limited concurrency, restart recovery, and result display
- Subscribe to Tauri events for task status, queue positions, and detail updates
- Coordinate the worker, ffmpeg/ffprobe, ONNX model, GPU sidecars, and runtime resources

## Commands

```bash
npm install
npm run desktop:dev
npm run typecheck
```

Windows runtime packaging:

```powershell
npm run desktop:windows:runtime
npm run desktop:windows:build
```

`desktop:windows:runtime` first exports `best.onnx` from `../ai-processor/weights/best.pt`, then builds the GPU sidecars, packages the Python worker, creates the CUDA runtime zip, and writes `src-tauri/resources/runtime/windows-x64/runtime-package-lock.json`. Build the runtime before the Windows installer so the app can enforce the matching runtime zip SHA256 during import. The Python worker build uses an isolated `.desktop-worker-venv` to keep conda training/inference dependencies out of the packaged worker.

macOS release commands must be run from `frontend/`:

```bash
npm run desktop:macos:ad-hoc
npm run desktop:macos:release-local
npm run desktop:macos:release-public
```

Do not mix `tauri dev`, raw `tauri build`, and the formal macOS release scripts. The trusted release entrypoint is `scripts/macos-release.mjs` and the matching npm scripts.

## Key Files

- `app/pages/index.vue`: batch import, task creation, task table, queue actions
- `app/pages/tasks/[id].vue`: task details, video preview, analysis results
- `app/app.vue`: global header, app settings, restart recovery, close confirmation
- `app/composables/useTaskApi.ts`: Tauri command API wrapper
- `app/composables/useTauriEvents.ts`: Tauri event subscription wrapper for task status and detail updates
- `src-tauri/src/lib.rs`: desktop database, queue scheduler, worker management, file management
- `scripts/macos-release.mjs`: macOS release orchestration
- `scripts/export-onnx-model.mjs`: export `best.pt` to `best.onnx`
- `scripts/build-gpu-sidecars.mjs`: build the C++ GPU preprocessing and ONNX CUDA analyzer sidecars
- `scripts/build-desktop-worker.mjs`: package the Python worker through `.desktop-worker-venv`
- `scripts/build-windows-runtime-package.mjs`: generate the Windows CUDA runtime zip

## Packaging Resource Rule

`src-tauri/resources/runtime/` and `src-tauri/resources/models/` are generated by `npm run desktop:windows:runtime` / `npm run desktop:build-worker` and must not be committed, except `src-tauri/resources/runtime/windows-x64/runtime-package-lock.json`. The lock file records the approved runtime zip size and SHA256 and is packaged into the Windows installer. The source repository also tracks `src-tauri/resources/placeholder.txt` to keep the directory structure.
